Binary Trees

General:

  • Rooted, ordered tree

  • Node can have two children; “leaf” nodes

  • Parent node is “root”; terminal node(s) are of bottomost level

  • Key is value corresponding to node

Searching:

  • Call ifNodeExists( Node node, int key ), checking if node, then if left or right of such is key recursively

  • Make node class

  • Check by using node’s key (node.data)

  • Check if node == null/key (ret false/true)

Heaps:

  • Completed binary tree

  • All nodes have two children; if have one, the child must be as left as possible

  • Is a Min/Max Heap

  • Value of root is smallest/largest of tree, and each parent node is smaller/larger than children nodes

Traversal:

  • Used to access part of tree

  • In-order: Left subtree, Root, Right subtree

  • Pre-order: Root, Left subtree, Right subtree

  • Post-order: Left subtree, Right subtree, Root

  • Root node visited last

Expression:

Inheritance

Abstract Classes

  • Can’t be instantiated

  • Parent several related classes with shared implementations

  • I.e. parts of a maze being under abstract class MazeComponent

  • Subclass (abstract or not) can only extend one abstract class

  • Have methods (abstract OR concrete) and variables

  • Abstract methods MUST be implemented in subclass if subclass is not abstract, as they are not implemented in parent abstract class

  • Have constructors (if subclass wants to use)

Inheritance

  • Child classes of parent extend them; “is-a” relationship

  • Has access to public parent methods and all of its own, but only parent fields through accessor methods

  • Can use super keyword to call parent methods (helpful if wishing to call an overridden method in child class), and this keyword for own fields/methods (both only within children classes themselves)

  • As children only have access to own fields without methods, if children have fields with same names in parent class, they only have access to their own field (intuitive)

  • Parent/superclass can have multiple children/subclasses, child/subclass can only have one parent/superclass

Constructors:

  • super(/*can be variables here*/) must be first call in child’s constructor

  • If called with variables aligning to parent initialization constructor, and own fields initialized, it’s fine

  • If not called (and child’s other fields initialized instead), java implicitly calls parent default constructor

  • Parent doesn’t have a default constructor but has initialization constructors → error

  • Parent has NO constructors, fields → null or 0

  • In the end, all of the child’s fields, including the parent’s fields, must be given a value

Interfaces

  • Can’t be instantiated, no constructors

  • Contain only method signatures, fields, and default methods

  • Default methods are like any method, and can be overridden

  • Implementing class(es) must implement all methods, and can implement multiple interfaces

Polymorphism

  • Used by casting subclass → parent class

  • Object, when instantiated, only has access to parent class methods and methods overridden by child

  • Overridden means same signature; overloaded is same name, different parameters; latter not accessible to polymorphic object

  • Allows the “instantiation” of interfaces

  • There are more than 4 bits

  • Left shift (<<): x << y shifts bits of x by y places to the left, inserting 0s

  • Same as multiplying by 2^y

None:

Quantifiers

  • place AFTER characters for clarity if [ ] or ( ) not used

  • If in between ANYTHING, it applies to BOTH

  • “aacc”.matches(“a.*c”) → true

  • “aabbcc”.matches(“[ab]\\w*c) → true

  • .*[ab] AND [ab].*→ >=0 occurrences of both

  • If if there is no group, it applies ONLY to the character to the right, and ALWAYS applies to contiguous groups

  • “aaabbb”.matches(“.*[a]b”) → false

  • “aaabbb”.matches(“\\w*ab”) → false

  • “aaabbb”.matches(“.*[ab]”) → true

  • Order matters without groups

  • “ccaa”.matches(“a\\w*c”) → false

  • “ccaa”.matches(“c\\w*a”) → true

  • “ccaa”.matches(“\\w*[ac]”) → true

  • any of specified a, b, or c

  • Usually used with ".*" → one of each character in string much match one in regex

  • Each character in abc doesn't have to match to one in string, as long as each in string matches to one in abc

[^abc] :

  • ^ applies to whole string without grouping

  • any besides specified a, b, or c

  • if a single character matches but the rest do not, still false

[a-zA-Z] :

  • any alphabetic character

[] :

  • necessary to group ranges, specify actual characters, etc.

  • i.e. “[.]” literally matches a . character

  • match String to String: "string1".matches(".*string2") or .matches (“.*[string2]”);

  • .matches("string2") or .matches("[string2]") fails

() :

  • groups different expressions into one regex (not necessary)

  • [a-zA-Z] works without it

{} :

  • denotes number of characters to match

  • Two integers matches from first to second int (inclusive)

| :

  • Works as thought, can have multiple

Methods
  • Must assign variable to result of methods (new objects created)

  • matches(String regex)

  • returns boolean

  • returns true if any string matches regex

  • if normal string input as regex (without []{}), strings must be exactly the same/chars align with other string exactly

  • “door”.matches(“d..r”) → T

  • replace()

  • returns String

  • replace(char oldChar, char newChar)

  • instances of oldChar→ newChar

  • replace(String target, String replacement)

  • instance of target→ replacement

  • replaceAll(String regex, String replacement)

  • returns String

  • matches of regex→ replacement

  • split()

  • returns String[ ]

  • Adding single character in regex splits at character

  • If splitting at special regex character, escape twice (split at periods → split(“\\.”) )

  • split(String regex)

  • splits string at instances of regex

  • split(String regex, int limit)

  • splits string at instances of regex into array of limit elements/length

  • works left to right; when potential array is at limit-1 elements, the rest of string, untouched, is added as last element

  • if limit is negative or 0, pretend the limit isn't there

  • Matcher m = p.matcher("aaaaab");

  • System.out.println(b); → true

  • System.out.println("aa".matches(".[ab]")); → true

  • System.out.println("BBAAssbbddll".matches("BBAAssbbddll.*")); → true

Searching

Binary

  • Assumes sorted array

  • Divides array in half

  • Divides left or right half depending on given

  • If given > middle index, divide right half into two

  • If given < middle index, divide left into two

  • Continue this recursively until down to one element; return this index or -1 if not found

Sequential

  • Extremely simple; start from left and compare elements to given; if match return index you’re at, or -1 if not found

Scanners

  • nextLine()

  • Reads whole next line into string

  • “\n” character included; can cause errors

  • next() 

  • Reads next String separated by space

  • nextInt(), nextDouble(), etc.

  • Get multiple numbers

  • If NOT using scan.nextLine(), use a dummy scan.nextLine() after end of each line to process "\n" character (not needed, but good practice)

Sorting

Arrays.sort()

  • Uses quicksort for primitives, mergesort for Comparators/Comparable

Collections.sort()

  • Uses mergesort; sorts any list of objects (as long as there is a compareTo() method given)

        Insertion

  • Sorts as one does playing cards

  • takes the element at index i and searches through the first i elements that have been sorted to find where to place it, moving the elements as it searches

MergeSort

  • Halves and halves array into their individual parts, then slowly merges them back in order

Quicksort

  • Chooses partition/split/dividing point (doesn’t matter which)

  • Partitions/breaks up array here, recursively calling partition on the point and comparing the elements

  • Element recombined at end

Selection Sort

  • Repeatedly searches through entire array for minimum element and puts it at the beginning/after previous minimum elements
